# 项目规划书

## 项目名称

基于 DeepSeek 的 Linux 运维诊断 Agent Skills 系统

## 项目目标

实现一个可以在 Linux 环境运行的自动化运维诊断工具。用户输入自然语言问题后，系统选择合适的运维 skill，执行受控 Shell 脚本，并生成诊断报告。

## 项目功能

- 磁盘空间诊断
- SSH 登录失败和系统日志分析
- 进程、CPU、内存状态检查
- 网络监听端口和连通性检查
- 自动化巡检报告生成
- DeepSeek 辅助意图识别和报告总结

## 使用技术

- Python
- Bash
- Linux 系统命令
- grep、sed、awk
- 管道与重定向
- 日志分析
- Agent Skills
- DeepSeek API

## 小组分工

- Linux 环境与脚本开发：负责 Shell 脚本、命令调试、权限检查。
- Python 功能实现：负责 CLI、skill 加载、安全执行器。
- AI Agent 接入：负责 DeepSeek 客户端、意图识别、报告总结。
- 测试与文档：负责测试用例、运行截图、项目报告和演示材料。

## 开发计划

1. 第 1 周：完成需求分析、项目结构、基础 Shell 脚本。
2. 第 2 周：完成 Python CLI、skill 加载、安全执行。
3. 第 3 周：接入 DeepSeek，完成报告生成和测试。
4. 第 4 周：准备答辩 PPT、演示视频、项目报告。

## 可能风险

- DeepSeek API 网络不可用：提供本地关键词匹配退化模式。
- 日志文件路径不同：兼容 `/var/log/auth.log`、`/var/log/secure` 和 `journalctl`。
- 普通用户权限不足：输出提示，不强制使用 sudo。
- 现场演示环境变化：保留报告、截图和演示视频作为备用。
