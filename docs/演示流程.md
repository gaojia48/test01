# 答辩演示流程

本文档用于项目答辩或录制演示视频。建议演示时控制在 3 到 5 分钟。

## 1. 准备环境

进入项目目录并激活虚拟环境：

```bash
cd ~/LinuxProject/WorkSpace/linux-ops-agent
source .venv/bin/activate
```

先做环境自检：

```bash
./ops doctor
```

讲解要点：

- Python、DeepSeek API Key、Linux 命令和脚本权限都由 `doctor` 检查。
- 如果 DeepSeek 不可用，项目仍可用本地关键词模式运行。

## 2. 展示可用 Skills

进入交互模式：

```bash
./ops
```

输入：

```text
list
```

讲解要点：

- 每个 skill 表示一个受控运维能力。
- AI 只能选择这些预先定义的 skill，不能直接执行任意 Shell 命令。

## 3. 演示磁盘检查

输入：

```text
检查磁盘空间问题
```

观察输出：

```text
选择方式：deepseek
执行 skills：disk_check
```

讲解要点：

- DeepSeek 负责理解自然语言并选择 `disk_check`。
- 本地脚本执行 `df`、`du`、`find` 等 Linux 命令。
- DeepSeek 根据脚本输出生成中文诊断报告。

## 4. 演示日志分析

输入：

```text
分析最近 SSH 登录失败
```

讲解要点：

- 日志分析 skill 会检查 `/var/log/auth.log`、`/var/log/secure` 或 `journalctl`。
- 使用 `grep`、`sed`、`sort`、`uniq` 等命令统计失败登录和系统错误。

## 5. 演示无 Skill 请求

输入：

```text
0
```

预期结果：

```text
执行 skills：无
```

讲解要点：

- 不明确的请求不会默认执行巡检。
- DeepSeek 会正常回答，不会误触发 Shell 脚本。

## 6. 查看和清理报告

查看报告列表：

```text
reports
```

查看最新报告：

```text
last report
```

清理报告：

```text
clear reports
```

讲解要点：

- 每次诊断都会在 `reports/` 下生成 Markdown 报告。
- `last report` 方便答辩时展示最近一次诊断结果。

## 7. 退出

输入：

```text
exit
```

## 备用演示命令

如果 DeepSeek 网络不稳定，可使用本地模式：

```bash
./ops 检查磁盘空间问题 --no-llm
./ops 分析最近 SSH 登录失败 --no-llm
./ops --no-llm
```

本地模式会显示：

```text
选择方式：keyword
```

## 答辩总结词

本项目是一个 Linux 运维 Agent Skills 系统。用户输入自然语言问题后，DeepSeek 负责识别意图并选择预设 skill，本地程序安全执行对应 Shell 脚本，最后由 DeepSeek 根据 Linux 命令输出生成诊断报告。项目重点体现了 Linux 命令、Shell 自动化、日志分析、Python 调用系统功能、Agent Skills 设计和 AI 工具辅助运维。
